# 优化复查记录（2026-07-12）

## 本轮已落地

1. 管理令牌由长期 `localStorage` 改为当前标签页 `sessionStorage`，并清理旧令牌、增加手动清除入口。
2. 自动监控增加数据质量闸门：周末、非当日行情、指数/候选日期不一致、K线失败率超过阈值时保留旧监控池，不自动覆盖。
3. 首页展示数据完整度与质量检查原因。
4. “评分”明确为“规则评分”，“正收益占比”明确为“观察期正收益比例”；少于30个可观察样本时标记为样本不足。
5. 多战法共振加分改为按独立战法大类计算，同类相关战法不重复加分。
6. HTTP 服务增加并发请求上限，超限返回 503，降低慢连接耗尽线程的风险。
7. 扫描运行状态的后台写入和页面读取统一加锁，减少状态竞态。
8. 新增 Python `unittest`，覆盖代码清洗、战法分类、共振加分、数据质量闸门和会话令牌。
9. 早盘页新增风险预算与仓位预案：按账户资金、单笔风险、单股仓位上限计算整手股数，并明确非“可计划内执行”状态只作预案。
10. 新增最近开盘核验轨迹与 `/api/opening/history` 接口，可复核定时任务是否按时运行及早盘结论变化。
11. 新增当前核验 CSV 导出，包含规则状态、计划价格、仓位和止损风险，并对 `= + - @` 开头字段做公式注入防护。
12. 删除“胜率优先”残留文案，避免把保守规则模型描述为收益或概率保证；测试扩展到 10 项。

## 仍建议继续推进

- 用 Nginx Proxy Manager 配置域名、HTTPS、连接数限制和管理接口来源限制；
- 将内联 CSS/JS 拆为静态文件并使用 CSP nonce/hash，最终移除 `unsafe-inline`；
- 建立固定 1/3/5/10 交易日的滚动回测，加入沪深300/中证1000基准、滑点和交易成本；
- 增加申万行业/概念集中度控制，避免自动池同题材过度集中；
- 采用正式授权行情源并记录每只股票实际数据源、延迟和完整度；
- 多实例部署时将限速、任务状态和缓存迁移到 Redis/数据库。
