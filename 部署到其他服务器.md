# 部署到其他 Linux 服务器

## 一、服务器要求

- Ubuntu 20.04+/Debian 11+（其他带 systemd 的 Linux 也可）
- Python 3.9 或更高版本
- 服务器能够访问互联网行情接口
- 默认开放 TCP 端口 `8766`
- 建议至少 2 核 CPU、4GB 内存；全市场扫描建议 4 核/8GB

## 二、Windows 一键上传部署

在本目录打开 PowerShell：

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\deploy_to_server.ps1 -HostName 你的服务器IP -User root
```

SSH 不是 22 端口时：

```powershell
.\deploy_to_server.ps1 -HostName 你的服务器IP -User root -SshPort 2222
```

脚本会上传源码到 `/opt/a_share_quant_web`，创建 Python 虚拟环境，安装依赖，并注册 `quant-web` systemd 服务。

## 三、手动部署

先从 Windows 上传文件：

```powershell
scp -r .\* root@服务器IP:/opt/a_share_quant_web/
```

登录服务器后执行：

```bash
cd /opt/a_share_quant_web
chmod +x deploy_on_server.sh
./deploy_on_server.sh
```

访问：

```text
http://服务器IP:8766/
```

## 四、防火墙和云安全组

Ubuntu UFW：

```bash
ufw allow 8766/tcp
```

如果是阿里云、腾讯云、华为云等，还要在云控制台安全组中放行入方向 TCP `8766`。生产使用建议配置域名、Nginx 和 HTTPS，而不是长期直接暴露 HTTP 管理接口。

## 五、管理令牌与运行数据

管理令牌和运行参数统一保存在：

```text
/etc/quant-web.env
```

权限必须保持为 `root:root 600`，不要把真实令牌写进源码、systemd 服务文件或公开代码仓库。运行数据单独存放在：

```text
/var/lib/quant-web
```

服务使用无登录权限的 `quantweb` 账户运行，源码仍由 root 只读管理。修改环境配置后执行：

```bash
systemctl restart quant-web
```

## 六、常用运维命令

```bash
systemctl status quant-web
systemctl restart quant-web
systemctl stop quant-web
journalctl -u quant-web -f
```

检查程序语法：

```bash
/opt/a_share_quant_web/.venv/bin/python -m py_compile /opt/a_share_quant_web/app.py
/opt/a_share_quant_web/.venv/bin/python -m py_compile /opt/a_share_quant_web/a_share_daily.py
```

## 七、后续更新源码

先备份，再上传替换：

```bash
cp /opt/a_share_quant_web/app.py /opt/a_share_quant_web/app.py.bak
```

Windows 上传：

```powershell
scp .\app.py .\a_share_daily.py root@服务器IP:/opt/a_share_quant_web/
```

服务器重启：

```bash
systemctl restart quant-web
systemctl is-active quant-web
```

## 八、需要保留的数据

迁移或更新时不要误删：

服务器上的运行数据位于 `/var/lib/quant-web/`：

- `a_share_daily_reports/`：历史扫描结果
- `opening_watchlist.json`：早盘监控清单
- `.a_share_cache/`：行情缓存，可删除后自动重建
- `logs/`：运行日志

源码位于 `/opt/a_share_quant_web/`，不要让运行账户拥有源码写权限。

## 九、定时任务

网站进程启动后会自动运行内部调度：

- 收盘扫描：默认工作日 `15:35,21:00`
- 开盘核验：默认工作日 `09:35,09:50,10:15,10:30`

具体时间可以在 `/etc/quant-web.env` 中通过环境变量调整。节假日会由实时行情日期再次校验，旧行情不会生成“可建仓”结论。
