# A股量化网站安全与逻辑复查记录（2026-07-11）

## 本轮逻辑修复

1. 自动监控最低评分由 66 提高到 72，边缘分候选不再自动进入次日池。
2. 自动池排除涨停情绪类、防守观察、高波动、换手过热、RSI过热、无有效买入区间或无止损的候选。
3. 自动池先限制同一战法分类最多 3 只，再按总分补足，降低单一风格集中。
4. 对历史/异常报告中的同代码重复记录进行合并，保留最高分方案并合并战法标签，避免重复占用监控名额。
5. 修正创业板/科创板“接近涨停”的判断阈值：20%涨跌幅股票不再沿用主板 8.5% 阈值。
6. 修正风险股票名称过滤：只判断 ST、*ST、N、C、退市等前缀，避免误伤名称中普通字母 N。
7. 源码目录与运行数据目录分离：源码 `/opt/a_share_quant_web`，数据 `/var/lib/quant-web`。

## 本轮安全修复

1. 服务从 root 改为无登录权限的 `quantweb` 低权限账户运行。
2. 源码由 root 只读管理，运行账户只能写 `/var/lib/quant-web`。
3. 管理令牌从 systemd 服务文件迁移到 `/etc/quant-web.env`，权限为 `root:root 600`。
4. 启用 systemd 沙箱：NoNewPrivileges、ProtectSystem=strict、ProtectHome、PrivateTmp、PrivateDevices、内核与命名空间限制等。
5. 基本面接口增加单 IP 与全局速率限制；开盘核验接口增加刷新限速。
6. 限速桶增加 4096 项硬上限，防止大量来源地址造成内存持续增长。
7. HTTP 连接增加 15 秒超时、后台请求线程自动回收、监听队列上限。
8. 拒绝写接口的 Transfer-Encoding，继续强制 JSON、Content-Length 和 20KB 请求上限。
9. 报告下载增加 10MB 大小上限并继续使用文件名白名单。
10. 增加 Cross-Origin-Resource-Policy 与 X-DNS-Prefetch-Control 安全响应头。
11. 基本面缓存保持固定容量，避免按股票代码无限增长。

## 尚需长期注意

- 网站仍通过 HTTP 8766 直接对公网提供访问。更稳妥的方案是只通过 Nginx Proxy Manager 的 HTTPS 域名访问，并在云防火墙中限制或关闭公网 8766。
- 行情源属于第三方公开接口，可能出现延迟、字段调整或临时不可用；系统已采用双源和旧行情日期保护，但不能保证数据永远无误。
- 量化评分是规则排序，不是收益概率；需持续累计至少数十到数百个交易样本，再按战法和市场环境做样本外评估。
